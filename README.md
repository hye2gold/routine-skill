# routine-buddy

Claude Code 안에서 작업 흐름과 어우러지게 루틴을 챙겨주는 리마인더 플러그인입니다.

물 마시기, 스트레칭, 업무일지, 미팅 같은 루틴을 등록해두면 별도 알림 앱을 띄우지 않아도 Claude Code 답변 끝에서 자연스럽게 챙겨줍니다.

> 알림 앱을 하나 더 보는 대신, 이미 일하고 있는 AI 대화 안에서 챙김을 받는 경험을 목표로 합니다.

## 왜 만들었나요?

하루 종일 업무를 하다 보면 작은 루틴을 놓치기 쉽습니다.

- 물을 마셔야 하는데 안 마심
- 오래 앉아 있는데 스트레칭을 안 함
- 업무일지나 회고를 까먹음
- 미팅이나 특정 일정 리마인더를 놓침

물론 별도 알림 앱을 둘 수도 있습니다. 하지만 이미 작업에 몰입하고 있으면 그 알림마저 닫아버리거나 못 보고 지나칠 수 있습니다.

routine-buddy는 알림을 밖에서 띄우는 대신, 사용자가 이미 보고 있는 Claude Code의 답변 흐름 안에 루틴을 살짝 끼워 넣습니다.

예를 들면 이런 식입니다.

```text
이 기획서는 문제 정의 다음에 성공 기준을 붙이면 좋아.
아참, 물 마실 시간이 됐어요. 지금 한 잔만 챙겨 마시면 좋아요 🥤
```

## 설치

Claude Code에서 아래 명령을 실행합니다.

```bash
/plugin marketplace add hye2gold/routine-skill
/plugin install routine-buddy@routine-buddy
```

설치 후 Claude Code를 재시작하면 훅이 활성화됩니다.

## 첫 실행 온보딩

설치 직후에는 한 번만 초기 세팅을 진행합니다.

1. 기본으로 추천할 말투를 물어봅니다.
2. 기본 루틴을 미리 세팅할지 물어봅니다.
3. 취소는 언제든지 가능하다는 점을 확인합니다.

온보딩 예시:

```text
설치 완료됐어요! 앞으로 루틴은 제가 Claude Code 답변 끝에서 자연스럽게 챙겨드릴게요.

어떤 말투를 원하시나요?
1. 따뜻하게 챙겨주는 말투(귀여운 이모지 포함)
2. 살짝 쪼아대는 긴급한 말투
3. 차분하고 담백한 말투

원하는 말투를 직접 말해도 괜찮아요. 말투는 언제든지 바꿀 수 있어요.
```

사용자가 답할 때까지 매번 같은 문장을 복붙하지 않고 조금씩 다른 표현으로 다시 물어봅니다. 여기서 고른 말투는 기본값으로 쓰이고, 실제 리마인더 말투는 새 루틴을 등록할 때마다 따로 고를 수 있습니다.

## 실제 사용 시나리오

### 물 마시기

사용자:

```text
/routine 물 2시간마다 알려줘
```

Claude:

```text
물 마시기는 어떤 말투로 챙겨줄까요?
1. 따뜻하게 챙겨주는 말투(귀여운 이모지 포함)
2. 조금 쪼아대는 긴급한 말투
3. 차분하고 담백한 말투
```

사용자:

```text
따뜻하게
```

Claude:

```text
오케이, 물 마시기는 오전 9시부터 오후 7시 사이에 2시간마다 따뜻하게 챙겨줄게.
```

이후 작업 중:

```text
이 API 설계는 인증 흐름을 먼저 분리해서 쓰면 더 읽기 좋아요.
아참, 물 마실 시간이 됐어요. 지금 한 잔만 챙겨 마시면 좋아요 🥤
```

사용자:

```text
마셨어
```

Claude는 해당 루틴을 완료 처리하고 다음 주기까지 조용해집니다.

### 오래 작업했을 때 스트레칭

사용자:

```text
/routine Claude Code 3시간째 쓰고 있으면 스트레칭 하라고 알려줘
```

이 루틴은 단순히 "오후 3시" 같은 고정 시간이 아니라, Claude Code에 프롬프트를 보내며 이어진 작업 흐름을 기준으로 합니다.

```text
여기서는 README보다 SKILL.md 쪽 문구를 먼저 맞추는 게 좋아요.
그리고 벌써 Claude Code 작업이 3시간째 이어지고 있어요. 어깨랑 목 한 번만 풀고 이어가요 🌿
```

### 특정날 미팅

사용자:

```text
/routine 내일 오후 4시 팀 미팅 잊지 마
```

Claude:

```text
이건 내일 하루만 알려줄까, 아니면 매일 반복할까?
```

특정 날짜 루틴은 헷갈리지 않도록 "그날만"인지 "반복"인지 먼저 확인합니다.

### 오전/오후가 애매할 때

사용자:

```text
/routine 내일 3시 미팅 알려줘
```

Claude:

```text
3시가 오전 3시야, 오후 3시야?
```

routine-buddy는 "3시"를 임의로 오전/오후로 추측하지 않습니다.

## 답변 스타일

루틴 알림은 Claude Code의 일반 답변 말투와 분리해서 설정할 수 있습니다. 말투는 전역 하나로 고정되는 것이 아니라 **루틴마다 따로 저장**됩니다.

예를 들어 물 마시기는 따뜻하게, 스트레칭은 긴급하게, 업무일지는 차분하게 설정할 수 있습니다.

| 스타일 | 예시 |
|---|---|
| 따뜻하게 | `물 마실 시간이 됐어요. 지금 한 잔만 챙겨 마시면 좋아요 🥤` |
| 긴급하게 | `물 마실 시간 지났어요. 지금 안 마시면 또 까먹습니다. 한 잔 갑시다.` |
| 차분하게 | `물 마실 시간이 됐습니다. 가능하면 지금 한 잔 드세요.` |
| 장난스럽게 | `물 타임입니다. 컵 들고 30초만 다녀옵시다.` |
| 직접 지정 | `친한 동료처럼 짧게 농담 섞어서 챙겨줘` |

말투 변경 예시:

```text
/routine 물 알림은 챙겨주는 듯한 따뜻한 말투에 귀여운 이모지도 같이 써줘
/routine 스트레칭 알림은 조금 쪼아대는 긴급한 말투로 해줘
/routine 업무일지 알림은 차분하고 담백하게 말해줘
```

## 설정할 수 있는 루틴 예시

| 하고 싶은 일 | 예시 멘트 |
|---|---|
| N분/시간마다 반복 | `/routine 물 2시간마다 알려줘` |
| 근무시간에만 반복 | `/routine 오전 9시부터 오후 7시까지만 물 2시간마다 알려줘` |
| 매일 특정 시각 | `/routine 업무일지 매일 오후 6시에 체크해줘` |
| 매주 특정 요일 | `/routine 매주 월,수,금 오전 10시에 스탠드업 알려줘` |
| 매달 특정 날짜 | `/routine 매달 25일 오전 9시에 월급 확인 알려줘` |
| 매년 특정 날짜 | `/routine 매년 3월 5일 오전 8시에 기념일 알려줘` |
| 특정날 1회 | `/routine 내일 오후 4시 팀 미팅 잊지 마` |
| Claude Code 연속 작업 | `/routine Claude Code 3시간째 쓰고 있으면 스트레칭 하라고 알려줘` |
| 여러 루틴 동시 등록 | `/routine 물은 2시간마다, 스트레칭은 매일 오후 4시에 알려줘` |
| 루틴별 말투 지정 | `/routine 물은 따뜻하게, 스트레칭은 긴급하게 챙겨줘` |
| 완료 처리 | `했어`, `마셨어`, `스트레칭 했어`, `확인했어` |
| 잠깐 미루기 | `30분 뒤에 알려줘`, `이따 알려줘`, `잠깐 꺼줘` |
| 완전 삭제 | `물 알림 취소해줘`, `스트레칭 알림 그만`, `미팅 알림 지워줘` |
| 목록 보기 | `등록된 루틴 보여줘`, `알림 목록 보여줘` |

## 어떻게 동작하나요?

routine-buddy는 백그라운드 알림 앱이 아닙니다. Claude Code에서 사용자가 메시지를 보낼 때마다 `UserPromptSubmit` 훅이 실행되고, 지금 챙길 루틴이 있는지 계산한 뒤 Claude의 답변 컨텍스트에 추가 정보를 넣습니다.

구성은 크게 세 부분입니다.

| 구성 | 역할 |
|---|---|
| `/routine` skill | 사용자의 자연어 요청을 루틴 등록/관리 명령으로 바꿉니다. |
| `hook.py` | 매 턴 실행되어 due 루틴과 온보딩 상태를 계산하고 Claude 답변에 컨텍스트를 주입합니다. |
| `manage.py` | 루틴 추가, 완료, 스누즈, 삭제, 말투 변경, 온보딩 상태 변경을 처리합니다. |
| `routines.json` | 루틴, 말투, 온보딩 상태를 저장하는 persistent memory입니다. |
| `session_state.json` | Claude Code 연속 작업 시간을 판단하기 위한 session memory입니다. |

즉 핵심은 이렇습니다.

```text
사용자가 Claude Code에 메시지를 보냄
→ hook.py가 루틴 저장소를 읽음
→ 지금 due인 루틴이 있는지 계산
→ 있으면 additionalContext로 Claude에게 전달
→ Claude가 원래 답변 끝에 자연스럽게 리마인드
```

## 저장 위치와 프라이버시

루틴 데이터는 로컬에 저장됩니다.

```text
~/.claude/routine-buddy/
```

due인 루틴의 label/id는 Claude가 답변 안에서 자연스럽게 챙길 수 있도록 해당 턴의 대화 컨텍스트에 포함됩니다.
