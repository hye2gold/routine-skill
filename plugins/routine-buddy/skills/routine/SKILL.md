# 루틴 비서 (routine)

직장인이 Claude Code로 일하는 도중에도, 주제와 상관없이 답변 속에 자연스럽게 끼어드는
리마인더를 등록·관리하는 스킬. 두 종류를 다룬다.

- **주기 루틴(recurring)**: "물 2시간마다", "한 시간마다 스트레칭" 등 — 매일 반복.
- **특정날 루틴(oneshot)**: "내일 4시 미팅", "모레 오전 약 복용" 등 — 지정한 날.

## 도구 경로 찾기 (중요 — 하드코딩 금지)

관리 CLI(`manage.py`)의 절대경로는 **훅이 매 턴 기록해 두는** 다음 파일에서 읽는다:

```
~/.claude/routine-buddy/cli.json     →  {"manage": "/abs/.../scripts/manage.py"}
```

먼저 그 경로를 구한다(택1):

```bash
# 권장: 훅이 기록한 cli.json에서 읽기
python3 -c "import json,os;print(json.load(open(os.path.expanduser('~/.claude/routine-buddy/cli.json')))['manage'])"
```

- 위 명령이 경로를 주면 그걸 `MANAGE` 로 삼아 아래 명령들을 실행한다.
- 만약 파일이 없으면(훅이 아직 안 돌았을 때), 플러그인 설치 위치의
  `${CLAUDE_PLUGIN_ROOT}/scripts/manage.py` 를 사용한다.

저장소는 `~/.claude/routine-buddy/routines.json` (전역 — 모든 세션 공유, 업데이트해도 보존).
오늘 날짜·현재 시각은 환경 컨텍스트(currentDate)나 `date` 명령으로 확인해 ISO로 환산할 것.

아래 예시들의 `$MANAGE` 는 위에서 구한 절대경로로 치환해서 실행한다.

---

## 1) 주기 루틴 등록

사용자가 "물 2시간마다 알려줘" 처럼 말하면:

```bash
python3 "$MANAGE" add --label "물 마시기" --type recurring --interval 120 --active-hours 9-19
```

- `--interval`: 분 단위 (2시간 = 120).
- `--active-hours`: 알림 허용 시간대. **직장인 기본값은 `9-19`** (근무시간에만 알림).
  사용자가 "자는 시간엔 빼줘" 류로 말하면 그 범위만. 24시간 원하면 옵션 생략.
- 등록 직후엔 한 주기가 지나야 첫 알림이 뜬다(바로 안 보채도 정상).

### 타입 고르기 (중요) — 6종

| 사용자 표현 | 타입 | 예시 명령 |
|---|---|---|
| "물 2시간마다", "30분마다 ○○" | `recurring` | `--type recurring --interval 120 --active-hours 9-19` |
| "매일 10:30 ○○", "매일 저녁 6시 ○○" | `daily` | `--type daily --at 10:30` |
| "매주 월/수/금 ○○", "매주 금 17:00" | `weekly` | `--type weekly --weekday 월,수,금 --at 10:00` |
| "매달 25일 ○○", "매월 1일 ○○" | `monthly` | `--type monthly --day 25 --at 09:00` |
| "매년 3월 5일 ○○", "매년 ○○" | `yearly` | `--type yearly --month 3 --day 5 --at 08:00` |
| "내일 4시 미팅" (특정 날짜 1회) | `oneshot` | `--type oneshot --due 2026-06-13T16:00 ...` |

## 1-2) 시각 고정형 (daily / weekly / monthly / yearly)

전부 `--at HH:MM`(24시간제)을 받고, 추가 인자만 다르다:

```bash
# 매일 같은 시각
python3 "$MANAGE" add --label "업무일지" --type daily --at 18:00
# 매주 요일(복수 가능: 콤마)
python3 "$MANAGE" add --label "스탠드업" --type weekly --weekday 월,수,금 --at 10:00
# 매달 며칠
python3 "$MANAGE" add --label "월급 확인" --type monthly --day 25 --at 09:00
# 매년 월/일
python3 "$MANAGE" add --label "기념일" --type yearly --month 3 --day 5 --at 08:00
```

- `--weekday`: `월~일` 또는 `mon~sun`, 여러 개는 콤마(`월,수,금`).
- `--day`: 1~31. 그 달에 없는 날(예: 31)은 **그 달 마지막 날로 자동 보정**.
- 공통 동작: 발생일의 그 시각이 지나면 알림 → "했어" 하면 그 차례 종료(다음 발생에 다시). 시각 전·완료 후엔 조용.
- 등록 시 오늘 슬롯이 이미 지났으면 오늘은 건너뛰고 다음부터.

## 2) 특정날 루틴 등록 — ⚠️ 먼저 질문할 것

사용자가 **특정 날짜/요일을 타겟**으로 말하면(예: "내일 4시 미팅 잊지 마"),
등록 전에 **AskUserQuestion으로 반드시 한 번 물어본다**:

> "이거 **그날(내일)만** 알려줄까, 아니면 **매일** 알려줄까?"

- **그날만** → oneshot 으로 등록:
  ```bash
  python3 "$MANAGE" add --label "팀 미팅" --type oneshot \
    --due 2026-06-13T16:00 --remind-from 2026-06-13T09:00
  ```
  - `--due`: 이벤트 시각. `--remind-from`: 그날 언제부터 상기시킬지(기본=due와 동일).
  - 시각 없는 종일 항목이면 `--due`에 그날 아침 시각을 넣어도 됨.
- **매일** → recurring 으로 등록(매일 같은 시각 1회 알림이면 사용자와 한 번 더 합의해 interval/active-hours 환산).

날짜에 시각이 없으면("내일 미팅 있어") 몇 시인지 가볍게 되묻고 ISO로 환산해 등록.

## 3) 등록 후 안내

등록하면 짧게 확인해 준다. 예:
> "오케이, 물은 9~19시 사이 2시간마다 챙겨줄게. 내일 4시 미팅은 그날 아침부터 리마인드 ON 🙆"

이후엔 **매 답변에서 자동으로** 챙긴다(훅이 컨텍스트를 주입). 스킬을 다시 부를 필요 없음.

---

## 4) 훅이 주입되면 (런타임 동작 — 매우 중요)

매 턴 훅이 `[루틴 비서]`로 시작하는 컨텍스트를 넣어준다. 그러면:

1. **지금 하던 대화를 끊지 말 것.** 본래 답변을 먼저 하고, **맨 끝에 한 줄** 가볍게 끼운다.
   - 좋은 예: "...(원래 답변)... 아 그리고 — 물 마신 지 2시간 넘었어, 한 잔 ㄱ? 💧"
   - 나쁜 예: 답변 첫머리부터 리마인더 / 매 문장 반복 / 알림 티 나는 딱딱한 박스.
2. **평소 말투로.** 사용자 톤에 맞춰 캐주얼하게. 과장·스팸 금지, 답변당 한 번만.
3. **사용자가 응답하면 상태 갱신** (훅 컨텍스트에 적힌 절대경로 명령을 그대로 Bash 실행):
   - "했어 / 응 / 마셨어 / 확인했어" → `done <id>` (주기는 리셋, 특정날은 종료).
   - "이따 / 30분 뒤에 / 나중에" → `snooze <id> <분>`.
4. **응답·확인이 없으면** 다음 답변에서도 계속 가볍게 상기(사용자가 말할 때까지). 단, 매번 살짝 다른 표현으로.

## 5) 관리 명령

- 목록: `python3 "$MANAGE" list`
- 완료/리셋: `python3 "$MANAGE" done <id>`
- 미루기: `python3 "$MANAGE" snooze <id> <분>`
- 삭제: `python3 "$MANAGE" remove <id>`

사용자가 "물 알림 그만", "미팅 알림 지워" 라고 하면 `remove`로 정리.
"물 알림 잠깐 꺼줘" 류는 `snooze`로 임시 처리.
