# 루틴 비서 (routine)

직장인이 Claude Code로 일하는 도중에도, 지금 작업 흐름과 어우러지게 답변 속에서 챙겨주는
리마인더를 등록·관리하는 스킬. 외부 OS 알림이 아니라 Claude Code 대화 안에서 함께 일하며 챙기는 동작이다.
타입은 7종이며(전체는 아래 "타입 고르기" 표 참고), 대표적으로:

- **주기 루틴(recurring)**: "물 2시간마다", "한 시간마다 스트레칭" 등 — N분마다 반복.
- **시각 고정형(daily/weekly/monthly/yearly)**: "매일 18:00", "매주 월·수·금", "매달 25일", "매년 3/5".
- **특정날 루틴(oneshot)**: "내일 4시 미팅" 등 — 지정한 날 1회(due 후 24시간 지나면 자동으로 조용).
- **작업 세션 루틴(session)**: "Claude Code 켜고 3시간째 일하면 스트레칭" 등 — Claude Code 대화 활동이 이어진 시간 기준.

## 도구 경로 찾기 (중요 — 하드코딩 금지)

관리 CLI(`manage.py`)의 절대경로는 **훅이 매 턴 기록해 두는** 다음 파일에서 읽는다:

```
~/.claude/routine-buddy/cli.json     →  {"manage": "/abs/.../scripts/manage.py"}
```

먼저 그 경로를 구한다(택1):

```bash
# 권장: 훅이 기록한 cli.json에서 읽기
python3 -c "import json,os;print(json.load(open(os.path.expanduser('~/.claude/routine-buddy/cli.json')))['manage'])"
```

- 위 명령이 경로를 주면 그걸 `MANAGE` 로 삼아 아래 명령들을 실행한다.
- 만약 파일이 없으면(훅이 아직 안 돌았을 때), 플러그인 설치 위치의
  `${CLAUDE_PLUGIN_ROOT}/scripts/manage.py` 를 사용한다.
- 그래도 불안하면 `python3 "$MANAGE" init` 을 한 번 실행하면 저장소와 `cli.json` 이 초기화된다.

저장소는 `~/.claude/routine-buddy/routines.json` (전역 — 모든 세션 공유, 업데이트해도 보존).
오늘 날짜·현재 시각은 환경 컨텍스트(currentDate)나 `date` 명령으로 확인해 ISO로 환산할 것.

아래 예시들의 `$MANAGE` 는 위에서 구한 절대경로로 치환해서 실행한다.

## 0) 설치 직후 온보딩

routine-buddy는 설치 직후 한 번만 초기 세팅을 진행한다. 훅이 `[루틴 비서 온보딩]` 컨텍스트를 주입하면, 사용자의 원래 요청에 답한 뒤 맨 끝에서 온보딩 질문을 이어간다.

**언제든 건너뛰기**: 사용자가 "지금은 안 할래", "나중에", "그만", "건너뛰기"처럼 온보딩 자체를 멈추고 싶어 하면, 어느 단계든 `python3 "$MANAGE" onboarding skip` 을 실행해 즉시 완료 처리하고 더 이상 묻지 않는다. (강제로 끝까지 진행시키지 말 것.)

온보딩 순서:

1. 말투 선택
   - 먼저 "설치 완료됐어요!"라고 말한다.
   - "어떤 말투를 원하나요?"라고 묻고 추천 예시 3개를 제시한다.
   - 추천 예시: 따뜻하게 챙겨주는 말투(귀여운 이모지 포함) / 살짝 쪼아대는 긴급한 말투 / 차분하고 담백한 말투.
   - 마지막에 "말투는 언제든지 바꿀 수 있어요"라고 덧붙인다.
   - 답변 처리:
     ```bash
     python3 "$MANAGE" onboarding tone --tone warm
     python3 "$MANAGE" onboarding tone --tone urgent
     python3 "$MANAGE" onboarding tone --tone calm
     python3 "$MANAGE" onboarding tone --custom "사용자가 말한 직접 말투"
     ```
2. 미리 세팅 여부
   - "미리 한 번 세팅해두실래요?"라고 묻는다.
   - 예/응/좋아:
     ```bash
     python3 "$MANAGE" onboarding preseed --answer yes
     ```
   - 아니오/괜찮아/나중에:
     ```bash
     python3 "$MANAGE" onboarding preseed --answer no
     ```
3. 미리 루틴 등록
   - 사용자가 예라고 하면 원하는 루틴을 받아 기존 등록 규칙대로 등록한다.
   - 최소 한 개를 등록했거나 사용자가 건너뛰겠다고 하면:
     ```bash
     python3 "$MANAGE" onboarding preseed-done
     ```
4. 취소 안내 확인
   - "취소는 언제든지 할 수 있어요. 확인하셨나요? 예/아니오로 답해주세요."라고 묻는다.
   - 예/응/확인:
     ```bash
     python3 "$MANAGE" onboarding cancel-ack --answer yes
     ```
   - 아니오:
     ```bash
     python3 "$MANAGE" onboarding cancel-ack --answer no
     ```
   - 사용자가 예/아니오로 답할 때까지 계속 묻는다.

온보딩 질문은 매번 같은 문장을 복붙하지 말고 살짝 다르게 말한다. 말투가 설정된 뒤에는 온보딩 질문도 그 말투를 따른다.
온보딩 말투는 새 루틴 등록 시 기본 추천값으로만 쓴다. 실제 리마인더 말투는 루틴마다 따로 저장한다.

## 1) 주기 루틴 등록

사용자가 "물 2시간마다 알려줘" 처럼 말하면:

```bash
python3 "$MANAGE" add --label "물 마시기" --type recurring --interval 120 --active-hours 9-19 --tone warm
```

- `--interval`: 분 단위 (2시간 = 120).
- `--active-hours`: 알림 허용 시간대. **직장인 기본값은 `9-19`** (근무시간에만 알림).
  사용자가 "자는 시간엔 빼줘" 류로 말하면 그 범위만. 24시간 원하면 옵션 생략.
- `--tone` / `--custom-tone`: **이 루틴 알림에만 적용되는 말투**. 루틴 등록 때마다 사용자가 어떤 말투로 챙겨주길 원하는지 확인해 함께 저장한다.
- 등록 직후엔 한 주기가 지나야 첫 알림이 뜬다(바로 안 보채도 정상).

### 루틴별 말투 선택 (등록할 때마다)

새 루틴을 등록할 때는 시간/주기 확인과 함께 "이 루틴은 어떤 말투로 챙겨줄까?"를 짧게 묻는다.

추천 예시:

- 따뜻하게 챙겨주는 말투(귀여운 이모지 포함) → `--tone warm`
- 조금 쪼아대는 긴급한 말투 → `--tone urgent`
- 차분하고 담백한 말투 → `--tone calm`

사용자가 "그냥 자연스럽게"라고 하면 `--tone natural`, "장난스럽게"라고 하면 `--tone playful`을 쓴다. 원하는 표현이 프리셋에 없으면 `--custom-tone "사용자가 말한 말투"`로 저장한다.

사용자가 말투를 명시하지 않았고 빨리 등록하길 원하면 온보딩에서 고른 기본 말투를 fallback으로 써도 된다. 그래도 가능하면 새 루틴 등록 확인 문장에 "이 루틴은 따뜻한 말투로 챙겨줄게"처럼 어떤 말투가 저장됐는지 알려준다.

### 타입 고르기 (중요) — 6종

| 사용자 표현 | 타입 | 예시 명령 |
|---|---|---|
| "물 2시간마다", "30분마다 ○○" | `recurring` | `--type recurring --interval 120 --active-hours 9-19` |
| "클코 3시간째 일하면 스트레칭", "일 시작하고 3시간째 알려줘" | `session` | `--type session --after 180 --idle-reset 45` |
| "매일 10:30 ○○", "매일 저녁 6시 ○○" | `daily` | `--type daily --at 10:30` |
| "매주 월/수/금 ○○", "매주 금 17:00" | `weekly` | `--type weekly --weekday 월,수,금 --at 10:00` |
| "매달 25일 ○○", "매월 1일 ○○" | `monthly` | `--type monthly --day 25 --at 09:00` |
| "매년 3월 5일 ○○", "매년 ○○" | `yearly` | `--type yearly --month 3 --day 5 --at 08:00` |
| "내일 4시 미팅" (특정 날짜 1회) | `oneshot` | `--type oneshot --due 2026-06-13T16:00 ...` |

### 여러 루틴을 한 번에 요청한 경우

사용자가 한 문장에 여러 루틴을 말하면 각각 독립 루틴으로 분리해서 등록한다.

예: "물은 2시간마다, 스트레칭은 매일 오후 4시에 알려줘"

```bash
python3 "$MANAGE" add --label "물 마시기" --type recurring --interval 120 --active-hours 9-19 --tone warm
python3 "$MANAGE" add --label "스트레칭" --type daily --at 16:00 --tone urgent
```

등록 후에는 등록된 항목과 각 말투를 한 번에 요약해서 알려준다. 단, 각각의 루틴에 날짜/시각/말투 모호성이 있으면 등록 전에 필요한 확인 질문을 먼저 한다.

## 1-1) 작업 세션 루틴 (session)

사용자가 "Claude Code 켜고 3시간째 되면 스트레칭 알려줘", "클코 계속 쓰고 있으면 3시간마다 한번 스트레칭 물어봐"처럼 말하면 `session` 타입으로 등록한다.

```bash
python3 "$MANAGE" add --label "스트레칭" --type session --after 180 --idle-reset 45 --tone warm
```

- `--after`: Claude Code 작업 활동이 몇 분째 이어졌을 때 상기할지. 3시간 = 180.
- `--idle-reset`: 이만큼 새 프롬프트가 없으면 "작업이 끊겼다"고 보고 세션 시작 시각을 리셋한다. 기본 45분.
- 훅은 백그라운드 타이머가 아니라 `UserPromptSubmit` 때만 실행되므로, 정확히는 "앱이 켜진 시간"이 아니라 **Claude Code에 계속 프롬프트를 보내며 작업한 흐름**을 기준으로 한다.
- 사용자가 "했어/스트레칭 했어"라고 하면 `done <id>`로 처리되고, 그 시점부터 다시 새 작업 세션 카운트가 시작된다.
- 사용자가 "30분 뒤에"라고 하면 `snooze <id> 30`으로 미룬다.
- 45분 넘게 아무 프롬프트가 없다가 다시 대화하면 새 작업 세션으로 본다.
- 작업 세션 시간은 Claude Code 창 단위가 아니라 **전역(모든 창 공유)**으로 추적한다. 여러 창을 동시에 쓰면 "연속 작업"은 전체 활동 기준이 된다.

## 1-2) 시각 고정형 (daily / weekly / monthly / yearly)

전부 `--at HH:MM`(24시간제)을 받고, 추가 인자만 다르다:

```bash
# 매일 같은 시각
python3 "$MANAGE" add --label "업무일지" --type daily --at 18:00
# 매주 요일(복수 가능: 콤마)
python3 "$MANAGE" add --label "스탠드업" --type weekly --weekday 월,수,금 --at 10:00
# 매달 며칠
python3 "$MANAGE" add --label "월급 확인" --type monthly --day 25 --at 09:00
# 매년 월/일
python3 "$MANAGE" add --label "기념일" --type yearly --month 3 --day 5 --at 08:00
```

- `--weekday`: `월~일` 또는 `mon~sun`, 여러 개는 콤마(`월,수,금`).
- `--day`: 1~31. 그 달에 없는 날(예: 31)은 **그 달 마지막 날로 자동 보정**.
- 공통 동작: 발생일의 그 시각이 지나면 알림 → "했어" 하면 그 차례 종료(다음 발생에 다시). 시각 전·완료 후엔 조용.
- 등록 시 오늘 슬롯이 이미 지났으면 오늘은 건너뛰고 다음부터.

### 오전/오후 해석

사용자가 "3시", "4시 미팅"처럼 오전/오후를 생략하면 **자동으로 오전/오후를 추정하지 말고 등록 전에 반드시 한 번 확인**한다.

- 확인 질문 예: "3시가 오전 3시야, 오후 3시야?"
- 사용자가 "오전/아침/새벽 3시"라고 답하면 `03:00`
- 사용자가 "오후/낮 3시"라고 답하면 `15:00`
- 사용자가 "밤/저녁 8시"라고 답하면 `20:00`

오전/오후가 이미 명시되어 있거나 "아침/새벽/낮/저녁/밤"처럼 시간대가 분명한 표현이 있으면 확인 없이 24시간제로 환산한다.

## 2) 특정날 루틴 등록 — ⚠️ 먼저 질문할 것

사용자가 **특정 날짜/요일을 타겟**으로 말하면(예: "내일 4시 미팅 잊지 마"),
등록 전에 **AskUserQuestion으로 반드시 한 번 물어본다**:

> "이거 **그날(내일)만** 알려줄까, 아니면 **매일** 알려줄까?"

- **그날만** → oneshot 으로 등록:
  ```bash
  python3 "$MANAGE" add --label "팀 미팅" --type oneshot \
    --due 2026-06-13T16:00 --remind-from 2026-06-13T09:00 --tone calm
  ```
  - `--due`: 이벤트 시각. `--remind-from`: 그날 언제부터 상기시킬지.
  - `--remind-from` 을 생략하면 기본값은 due 당일 09:00(단, due가 09:00 이전이면 due 시각)이다.
  - 시각 없는 종일 항목이면 `--due`에 그날 아침 시각을 넣어도 됨.
- **매일** → recurring 으로 등록(매일 같은 시각 1회 알림이면 사용자와 한 번 더 합의해 interval/active-hours 환산).

날짜에 시각이 없으면("내일 미팅 있어") 몇 시인지 가볍게 되묻고 ISO로 환산해 등록.

## 3) 등록 후 안내

등록하면 짧게 확인해 준다. 예:
> "오케이, 물은 9~19시 사이 2시간마다 따뜻하게 챙겨줄게. 내일 4시 미팅은 차분한 말투로 그날 아침부터 리마인드 ON."

이후엔 due 상태가 되면 **매 답변에서 자동으로** 챙긴다(훅이 컨텍스트를 주입). 스킬을 다시 부를 필요 없음.
이 반복성은 의도된 제품 경험이다. 사용자가 작업 중이어도, 원래 답변을 먼저 완성한 뒤 자연스럽게 이어서 챙긴다.

일반 루틴 추가 과정에서는 온보딩을 다시 시작하지 않는다. 대신 말투는 **새 루틴을 등록할 때마다 루틴별로 저장**한다. 이미 등록된 루틴의 말투만 바꾸고 싶으면 `style --id <id>`를 사용한다.

## 4) 훅이 주입되면 (런타임 동작 — 매우 중요)

매 턴 훅이 `[루틴 비서]`로 시작하는 컨텍스트를 넣어준다. 그러면:

1. **지금 하던 대화를 끊지 말 것.** 본래 답변을 먼저 충분히 하고, 그 답변의 흐름과 어우러지게 **맨 끝에 한 번** 챙긴다.
   - 좋은 예: "...이 기획서는 문제 정의 다음에 성공 기준을 붙이면 좋아. 아참, 4시 루틴인데 물 한 잔 했어?"
   - 나쁜 예: 답변 첫머리부터 리마인더 / 매 문장 반복 / 알림 티 나는 딱딱한 박스.
2. **각 루틴별 말투로.** 훅 컨텍스트의 "이 루틴 말투"를 우선 따른다. 사용자 톤에 맞춰 캐주얼하게, 과장·스팸 금지, 답변당 한 번만.
3. **사용자가 응답하면 상태 갱신** (훅 컨텍스트에 적힌 완료명령/미루기명령을 그대로 Bash 실행):
   - "했어 / 응 / 마셨어 / 확인했어" → `done <id>` (주기는 리셋, 특정날은 종료).
   - "이따 / 30분 뒤에 / 나중에" → `snooze <id> <분>`.
4. 여러 루틴이 동시에 떠 있으면, 사용자가 어느 항목을 말하는지 분명할 때만 완료 처리한다. 애매하면 짧게 확인한다.
5. **응답·확인이 없으면** 다음 답변에서도 계속 가볍게 상기(사용자가 말할 때까지). 단, 매번 살짝 다른 표현으로.

## 5) 관리 명령

- 목록: `python3 "$MANAGE" list`
- 완료/리셋: `python3 "$MANAGE" done <id>`
- 미루기: `python3 "$MANAGE" snooze <id> <분>`
- 삭제: `python3 "$MANAGE" remove <id>`
- 기본 말투 확인/설정: `python3 "$MANAGE" style`
- 특정 루틴 말투 확인/설정: `python3 "$MANAGE" style --id <id>`

### 취소 / 삭제 / 잠깐 미루기

- 사용자가 "물 알림 취소", "물 알림 그만", "미팅 알림 지워"처럼 말하면 해당 루틴을 `remove`로 완전히 삭제한다.
- 삭제된 루틴은 더 이상 훅에 나타나지 않는다. 다시 받고 싶으면 새로 등록해야 한다.
- oneshot을 "했어/확인했어"라고 하면 `done`으로 active=false 처리되어 종료된다. 기록에는 남지만 다시 뜨지 않는다.
- "잠깐 꺼줘", "이따 알려줘", "30분 뒤에"처럼 임시 중단이면 삭제하지 말고 `snooze`로 미룬다.
- 어떤 루틴을 취소하는지 애매하면 먼저 목록을 보고 사용자에게 확인한다.

### 루틴 비서 말투 설정

말투는 새 루틴을 등록할 때마다 해당 루틴에 저장한다. 온보딩/기본 말투는 fallback일 뿐이고, due 알림에서는 각 루틴의 `tone_instruction`을 우선 사용한다. Claude Code 전체 말투를 바꾸는 것이 아니라 **routine-buddy 알림 문장에만 적용되는 전용 말투**다.

```bash
python3 "$MANAGE" style --tone warm
python3 "$MANAGE" style --tone urgent
python3 "$MANAGE" style --tone calm
python3 "$MANAGE" style --tone playful
python3 "$MANAGE" style --tone natural
python3 "$MANAGE" style --id 물-마시기 --tone urgent
python3 "$MANAGE" style --id 스트레칭 --custom "트레이너처럼 짧고 단호하게 말해줘"
```

지원 프리셋:

- `natural`: 평소 대화 흐름에 맞춰 자연스럽고 짧게 챙긴다.
- `warm`: 챙겨주는 듯한 따뜻한 말투로, 부담 주지 않고 다정하게 묻는다. 문장 끝에 귀여운 이모지를 가볍게 곁들인다.
- `urgent`: 조금 쪼아대는 긴급한 말투로, 지금 바로 하게 만드는 느낌을 준다.
- `calm`: 차분하고 담백한 말투로, 과장 없이 조용히 상기한다.
- `playful`: 가볍고 장난기 있는 말투로, 너무 딱딱하지 않게 챙긴다.

사용자가 원하는 말투가 프리셋에 없으면 직접 지시를 저장한다.

```bash
python3 "$MANAGE" style --custom "친한 동료처럼 짧게 농담 섞어서 챙겨줘"
```

현재 기본 말투를 묻거나 "루틴 기본 말투 뭐야?"라고 하면 `python3 "$MANAGE" style` 로 확인한다. 특정 루틴의 말투를 묻거나 바꾸면 `python3 "$MANAGE" style --id <id>`를 사용한다.
